📊 Why You Get 200–900 Leads Instead of 5,000 — And How We Fix It
❌ The Real Problem
The NLM government API is the only free browser-compatible healthcare API — it hard-caps at 7,500 records per search term and 500 per page. After applying state, phone, and zip filters you often get only 200–900 results matching your exact criteria.

The official NPPES registry (npiregistry.cms.hhs.gov) has 8+ million providers but blocks all browser requests (CORS) — it requires a paid server backend. No free workaround exists for it in a browser.
✅ Our Fix — A–Z Alphabet Sweep
We run 26 automatic searches: "Family Medicine A", "Family Medicine B" … "Family Medicine Z". Each letter returns a different slice of providers. We run both the Individuals API and Organizations API and deduplicate everything by NPI.

26 letters × 2 APIs × 7,500 = up to 390,000 potential records. For popular specialties like Family Med, Dentist, or Psychiatry you will consistently pull 5,000–25,000+ unique providers.
